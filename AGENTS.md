# AGENTS.md - SEFIN Audit Tool

This document provides guidelines and instructions for agents operating within the SEFIN Audit Tool codebase.

## 🏗️ Architecture & General Guidelines

*   **Hybrid Architecture**: The system uses a Node.js API Gateway (Express/tRPC) proxying to a Python backend (FastAPI/Polars) for heavy data processing.
*   **Initialization**: The project is initialized via `node start.js` or `python app.py`. These scripts automatically handle dependencies (`pnpm` and `pip`), database initialization (`init_db.ts`), and generating a minimal `.env` file if it's missing. The default local DB is SQLite (`DATABASE_URL=file:./sefin_audit.db`).
*   **Env Variables**: The `.env.example` file is strictly structured into '1. OBRIGATÓRIAS', '2. OPCIONAIS', '3. AVANÇADAS', and '4. LEGADAS'. Mandatory variables correspond exactly to the minimal `.env` automatically generated by the bootstrap scripts. Legacy configuration (PostgreSQL, PySide6) and Oracle database credentials must not be included in the generated `.env` main flow.
*   **Refactoring**: Prioritize incremental and controlled changes that preserve existing public endpoints and contracts. Avoid excessive architectural abstractions and circular imports. When refactoring utility functions (e.g., reducing parameters by introducing a dataclass), ensure backward compatibility for positional arguments or systematically identify and update all calls across the entire codebase. Limit code health improvements to specific targeted files.
*   **Artifact Statistics**: To retrieve summarized artifact statistics (file counts, last modification date) for an audit historic record, use `core.audit_artifacts_service.obter_estatisticas_arquivos`.
*   **Path Validation**: To prevent Path Traversal, path validation functions (like `is_safe_path`) must return the securely resolved `Path` object (e.g., `Path | None`), and calling code must strictly use this returned object rather than the original unvalidated path string.

## 🐍 Python (Data Processing & FastAPI)

*   **Engine**: Polars is the primary engine for data wrangling.
*   **Polars Filtering**: When filtering Polars DataFrames for boolean truthiness, use `pl.col('column_name')` directly instead of `pl.col('column_name') == True` to avoid `ruff` E712 linting errors.
*   **Polars Iteration**: Avoid converting Polars DataFrames to Python dictionaries using `to_dicts()` for row-wise processing. Utilize native Polars expressions instead. If Python iteration is strictly required, extract columns to native Python lists explicitly by name (e.g., `df.get_column('name').to_list()`) and iterate using `zip()`. Avoid positional unpacking of dynamically fetched columns based on external constants.
*   **CNPJ Parameters**: When calling or testing FastAPI routes with CNPJ path parameters (e.g., `/auditoria/status/{cnpj}`), strip punctuation and use only digits (e.g., '00000000000191'). Slashes will cause 404 errors.
*   **Audit Pipeline**: The pipeline orchestration is centralized in `server/python/core/audit_pipeline_service.py`. It persists real-time execution state in `status_pipeline.json`, which is consumed by the `/auditoria/status/{cnpj}` endpoint. Responses are centrally constructed using `core.audit_response_service.construir_resposta_status`.
*   **Config Loader**: The Python backend dynamically loads variables from a root `config.py` module using `core.config_loader.get_config_var()`. Do not use `importlib.util` blocks inside API routers.
*   **Type Hints**: Python 3.12+ natively supports generic type hints. Do not globally remove `from __future__ import annotations`. Only remove it if explicitly requested for a specific file.
*   **Text Normalization**: For high-performance text normalization in Python data pipelines, use precompiled regular expressions (`re.compile`) defined globally rather than inline `re.sub` calls, and prefer native string operations like `' '.join(text.split())` over regex for removing multiple consecutive whitespaces.
*   **Linting & Formatting**: Use `ruff` (version 0.15.5) for Python linting (`ruff check <file>`) and formatting (`ruff format <file>`). Do not use prettier for Python files. If an automated code reviewer blocks a PR due to the removal of an unused standard library import (like `import re`), restoring the harmless import is an acceptable workaround.

## 🌐 Frontend & Node.js

*   **Frameworks**: React 19, TypeScript, Vite, Tailwind CSS v4, and shadcn/ui.
*   **File Metadata**: When defining TypeScript interfaces for file metadata (e.g., `AuditFileResult`), make fields like `rows` and `columns` optional to prevent UI crashes if the Python backend cannot extract them (e.g., for non-Parquet files).
*   **CSS Injection**: To prevent XSS via CSS injection in React `<style>` tags, pass the dynamic CSS string as standard React children (`<style>{cssString}</style>`) rather than using `dangerouslySetInnerHTML`.
*   **Process Spawning**: When using `subprocess.Popen` to execute node package manager commands like `pnpm` in cross-platform Python scripts (e.g., `app.py`), explicitly handle the `.cmd` extension on Windows (e.g., `['pnpm.cmd' if sys.platform == 'win32' else 'pnpm', ...]`) when `shell=False`.
*   **Formatting**: Format only the modified files using `npx prettier --write <filepath>`. Do not run `pnpm format` globally.
*   **Package Managers**: Never modify `package.json` or `tsconfig.json` without explicit instruction.

## 🧪 Testing & Verification

*   **Test Suite execution**:
    *   **Frontend/Node.js tests**: `pnpm test` executes both frontend (`client/**/*.test.ts`) and backend (`server/**/*.test.ts`) unit tests via the central `vitest.config.ts`. Run type checking with `cd client && pnpm check`. If commands like `tsc` are not found, run `pnpm install` in both the root and `client/` directories first.
    *   **Python tests**: Execute from the project root using `PYTHONPATH=server/python python -m pytest server/python/ -m 'not e2e'`. For legacy Python utilities in `src/`, test by creating files in `tests/` and running `PYTHONPATH=src python -m pytest tests/`.
*   **Python API Contract Tests**: Use `fastapi.testclient.TestClient` and `unittest.mock.patch` to bypass database, file system, and dynamic config loading (e.g., patching `core.config_loader.get_config_var`). Explicitly assert the presence and data types of mandatory JSON response keys. If encountering a `ModuleNotFoundError` for `httpx`, install it manually (`pip install httpx`).
*   **Background Tasks**: When testing FastAPI endpoints using `BackgroundTasks`, mock execution by patching `fastapi.background.BackgroundTasks.add_task` (with `create=True`).
*   **Cross-platform testing**: When using `child_process.execSync` to execute Python tests from JS, pass `PYTHONPATH` via the `env` options object (`{ env: { ...process.env, PYTHONPATH: 'server/python' } }`).
*   **Local testing/Vite Dev Server**: The Vite dev server can be run standalone (`cd client && npx vite --port 3000`), but testing authenticated flows may block at the OAuth login screen.
*   **Playwright UI Verification**: To visually verify frontend UI changes locally, write a temporary Python Playwright script (using `playwright.sync_api`) to capture a video and screenshot. When navigating the 'Tabelas' (Parquet viewer) page locally, fill the directory input and simulate an `Enter` press (`input.press('Enter')`). For simple UI component changes where full authentication mocking is unavailable on standalone Vite, rely on manual review.
*   **Git Diff Review**: When reverting changes via Git (e.g., `git restore .`) before requesting a code review, ensure the staging area is completely cleared by running `git reset --hard HEAD`.
