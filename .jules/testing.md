## 2025-02-28 - Testing the Math.nan case correctly in _clean_value
**Learning:** When writing tests that involve explicit checks like `math.isnan()` added directly to the implementation based on a prompt snippet, you must verify the codebase correctly mirrors the intended functionality of that snippet. However, you should also be mindful of how imports and Python module paths are mocked. In my case, I mistakenly patched `server.python.core...` instead of `core...` when the `PYTHONPATH` was set to `server/python`, breaking tests.
**Action:** When using `unittest.mock.patch` in tests run with `PYTHONPATH=server/python`, the module path must be relative to the PYTHONPATH root (e.g., `patch('core.produto_runtime...')`), not the absolute repository path (e.g., `patch('server.python.core...')`). Also, align the implementation accurately to the task snippet to ensure correct test coverage logic.
