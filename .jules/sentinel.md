## 2024-04-03 - Path Traversal in File Uploads via FastAPI UploadFile.filename
**Vulnerability:** The endpoints handling file uploads (`/api/python/parquet/upload`) in `server/python/routers/parquet.py` and `server/python/api-Enio.py` used `file.filename` directly to construct the destination path (`file_path = dir_path / file.filename`). Because the filename is controlled by the client, it can contain path traversal characters (e.g., `../../../etc/passwd`), allowing an attacker to write files outside the intended directory.
**Learning:** Even when the target directory (`dir_path`) is validated and strictly resolved against allowed paths, concatenating an unsanitized `UploadFile.filename` bypasses those checks because `dir_path / file.filename` evaluates the traversal elements during path construction.
**Prevention:** Always sanitize the client-provided filename by extracting only its base name using `Path(file.filename).name` (or similar safe methods like `werkzeug.utils.secure_filename`) before appending it to a destination directory path.
