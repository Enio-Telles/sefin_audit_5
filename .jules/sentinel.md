## 2025-04-08 - Prevent Path Traversal in Parquet Upload
**Vulnerability:** The `upload_parquet` endpoint in `server/python/api-Enio.py` and `server/python/routers/parquet.py` used `file.filename` directly to construct the save path: `file_path = dir_path / file.filename`. This is a classic path traversal vulnerability, as `UploadFile.filename` is controlled by the user and can contain path components like `../../` to write files outside the intended directory.
**Learning:** Python/FastAPI file uploads do not automatically sanitize the filename provided by the client. Direct concatenation of user-provided filenames with backend directory paths is always unsafe, even if the directory path itself is validated, because the filename can traverse out of the validated directory.
**Prevention:** Always sanitize the filename before using it in path construction. A simple and effective method is extracting just the base name using `Path(file.filename).name`. This removes any preceding path components provided by a malicious client.
