## 2025-04-02 - Path Traversal Vulnerability in Excel Export Endpoints
**Vulnerability:** The `/api/python/export/excel` and `/api/python/export/excel-download` endpoints in `server/python/routers/export.py` accepted raw file paths (`request.output_dir`, `request.source_files`, `file_path`) and performed filesystem operations (`read_parquet`, `mkdir`) without validating if the resulting paths were outside the allowed project directories. This allowed arbitrary file reads (converting arbitrary system files to Excel) and arbitrary directory creation writes by a malicious actor supplying paths like `../../../../etc/passwd`.
**Learning:** Even though `_is_path_allowed` whitelist validation exists and is utilized correctly in `filesystem.py`, newly added endpoints dealing with files (like exports) often miss these validations if not explicitly enforced.
**Prevention:** All backend API endpoints handling file paths must import and use `_is_path_allowed()` from `server/python/routers/filesystem.py` to validate user input against the global directory whitelist *after* calling `.resolve()`.
