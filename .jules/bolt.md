## 2025-02-12 - [Polars Lazy Evaluation with Conditional Intermediary Caching]
**Learning:** Forcing a `.collect()` to measure the execution time of a LazyFrame breaks Polars' global query optimization (like predicate pushdown) because it splits the DAG. However, conditional requirements (e.g. caching *only* if the execution takes > 30s) demand an evaluation to measure the time.
**Action:** When a business rule requires conditional intermediate caching based on evaluation time, evaluate the `.collect()` eagerly, but if the condition isn't met (e.g. time < 30s), discard the eager DataFrame and return the *original* unevaluated `LazyFrame` expression. This elegantly satisfies both the caching constraint for heavy queries and preserves the single, holistically optimized DAG for fast queries. Also, always remember to wrap `pl.Categorical` operations across multiple datasets with a global `pl.enable_string_cache()`.
